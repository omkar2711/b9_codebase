react 
    - Virtual dom
    - JSX
    - Components : Class based and Functional
    - Props and State




props :  - Props are used to pass data from one component to another component.
  - Props are read-only.
  - Props are immutable.

state :  - State is used to manage data within a component.
  - State is mutable.
  - State can be changed using setState() method in class components or using useState() hook in functional components.


Hooks :  - Hooks are functions that allow you to use state and other React features in functional components.
  - Commonly used hooks are useState, useEffect, useContext, useReducer etc.

  useState :  - useState is a hook that allows you to add state to functional components.
  - It returns an array with two elements: the current state value and a function to update




  State : State is data that changes over time and affects the UI.

Problem prop Drilling:
    passing props multiple levels
    hard to manage & debug

Global State Management

Redux:
    Centralized state 
    Predectable behaviour
    Easy to maintain and debug
    flexible, larger apps

Redux Core principle
    1. Single Source of Truth
    2. State is Read Only
    3. Changes are made through Pure Functions(Reducers)

Redux:
    Too much code / boilerPlates
    Too many files to maintain


Redux Toolkit 
1. Store : Its a central place where the entire application's state is stored.  -> configureStore
2. Action : is an object describing what event had occourred, type
    Action Creator (function) : return an action object
3. Reducers : Pure Functions 
            Current/initial State 
            Action
            Returns -> Updated State / new State
4. Dipatch : Used to send action to the store -> useDispatch();
5. Selectors : Are functions used to extract the state from the store -> useSelector();

6. Slice : (State + action creators + reducers) -> createSlice
7. Thunk / createAsyncThunk : Async code / logic 
8. Provider : Connects store to react
